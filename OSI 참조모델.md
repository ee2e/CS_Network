# OSI 참조모델

## 1. 개요

> ISO 및 CCITT에서 권고하는 개발형 시스템 상호 접속(Open System Interconnection; OSI)을 위한 참조 모델
>
> OSI 참조 모델은 컴퓨터 통신망 구조의 기본 모델이며 7개의 계층으로 구성되어 있음

### 1.1 OSI 모델의 목적

- ISO 7498 (1983년)
- 개방형 시스템(의미 : 적용 가능한 표준을 상호 인정하고 지원)의 상호접속을 위한 참조 모델
- 이기종(heterogeneous) 시스템간의 통신이 가능하도록 함

- 인터페이스는 SAP(Service Access Point)로 불리며 각 계층에서 바로 위, 아래 계층에게 정보와 서비스를 제공



### 1.3 계층의 분리원칙

- 너무 많은 계층으로 분리함으로써 각 계층에 대한 설명과 이들에 대한 조합이 필요 이상으로 많지 않아야 함
- __서비스의 양이 적고 경계를 중심으로 최소의 상호 작용이 일어나도록 경계를 정해야함__
- 수행하는 일의 측면에서나 필요로 하는 기술의 측면에 있어서 명백히 서로 다른 기능들을 다룰 수 있도록 계층을 설정
- 비슷한 기능들은 같은 계층에 존재하도록 함
- 과거의 경험에 의해 성공적이라 판단되는 곳에 경계를 설정함
- 쉽게 세분화되는 기능을 하나의 계층으로 함.
- 필요한 경우 표준화된 인터페이스를 가질 수 있는 곳에 경계를 설정함
- 데이터의 조작, 즉 구문(syntax), 의미(semantic) 등의 추상적 개념에 대해 서로 다른 수준을 필요로 하는 곳에서 계층을 설정함
- __각 계층은 단지 상위 계층과 하위 계층에 경계를 갖도록 함.__ 즉, 다른 계층과는 전혀 무관하도록 함



### 1.4 각 계층이 필요한 이유

- 물리 계층 : 종단 간을 연결하려면 다양한 물리적 매체를 사용하는 구조가 필요
- 데이터링크 계층 : 전화회선, 광섬유 등의 물리적 통신 매체를 사용할 경우에 서로 다른 데이터링크 제어 절차가 필요
- 네트워크 계층 : 전송 주체(송신자, 수신자) 사이에 중간 노드가 있는 경우에도 전송 주체 간의 연결 통로(네트워크 연결)를 제공
- 전송 계층 : 발신지 시스템에서 목적지 시스템까지의 신뢰성 있는 데이터 이동을 제어
- 세션 계층 : 대화를 조직하고 동기화하거나 데이터 교환을 관리
- 표현 계층 : 응용 프로그램의 구조화된 데이터를 표현하고 조정하는 기능
- 응용 계층 : 응용 프로그램을 위한 프로토콜을 함께 관리



### 1.5 캡슐화

- 캡슐화 : 데이터에 각 계층의 제어정보를 추가하는 것
- Service Data Unit + Protocol Control Information => Protocol Data Unit



## 2. 응용 계층

> OSI 모델의 최상위 계층

### 2.1 목적

#### 응용 계층이 제공하는 서비스

- 공통 응용 서비스 요소(CASE : Common Application Service Element) : 응용의 종류에 관계없이 개방 시스템을 연결하는 방법을 제공
- 각 응용에 속하는 서비스 요소 : 응용에 따라서 제공되는 서비스로서 파일 전송, 데이터베이스의 접근, 텔넷, 전자우편 등과 같이 응용과 관련된 서비스 요소
- 각 사용자에 속하는 서비스 요소

### 2.2 응용 계층의 기능

- 응용 계층 기능들의 그룹화
- 응용 프로세스 관리
- 시스템 관리
- 계층관리



## 3. 표현 계층

> 응용 계층과 세션 계층 사이

### 3.1 목적

- 응용체들 사이의 데이터 교환
- 응용체들이 통신하거나 참조할 수 있도록 정보를 표현하는 기능을 제공
  - 응용체들 간에 교환되어야 할 데이터를 표현
  - 응용체들의 통신에서 참조할 수 있는 데이터 구조를 표현
- 구문독립성을 제공
  - 표현 계층은 데이터가 표현되는 __구문(syntax)에만 관계__하며, 응용 계층과 관계있는 의미(semantics)와는 무관함

### 3.2 표현 계층의 역할

- 구문변환 : 코드와 글자의 변환, 데이터 구조의 변환, 데이터 구조에 대한 연산의 결정 등을 포함
- 구문선택 : 초기에 알맞은 구문을 선택하고 선택한 구문을 수정하는 것을 포함
- 암호화
- 압축

### 3.3 표현 계층의 기능

- 구문의 변환
- 구문의 절충
- 주소지정과 다중화
- 표현 계층 관리



## 4. 세션 계층

> 표현 계층과 전송 계층 사이

### 4.1 목적

- 표현체들 사이의 데이터 교환
- 표현 계층들의 대화 세션의 형성과 동기, 정상적인 데이터의 교환

### 4.2 세션 계층의 역할

- 대화 제어
- 동기화

### 4.3 세션 계층의 기능

- 세션, 전송 연결 간의 대응
- 세션 연결의 흐름제어
- 신속 처리 데이터 전송
- 세션 연결 복구
- 세션 연결 해제
- 세션 계층관리



## 5. 전송 계층

> 세션 계층과 네트워크 계층 사이

### 5.1 목적

- 세션 계층들 사이의 데이터 교환
- 세션을 갖고 있는 두 이용자 사이의 정확한 데이터 전송 서비스를 제공

### 5.2 전송 계층의 역할

- 포트 주소지정
- 분할과 재조립
- 연결제어
- 흐름제어 및 오류제어

### 5.3 전송 계층의 기능

- 주소 변환(전송 주소를 네트워크 주소로 변환)
- 다중화와 분리
- 동작 단계(전송 연결의 설정과 해제)
- 전송 계층관리



## 6. 네트워크 계층

> 전송 계층과 데이터링크 계층 사이

### 6.1 목적

- 전송체들 사이의 데이터 교환
- __라우팅 전략__을 사용하여 데이터를 송신지 컴퓨터로부터 여러 네트워크를 거쳐 수신지 컴퓨터까지 전달

### 6.2 네트워크 계층의 역할

- 논리주소 지정
- 라우팅

### 6.3 네트워크 계층의 기능

- 라우팅과 중계
- 네트워크 연결
- 네트워크 연결의 다중화
- 세그먼팅과 블로킹
- 오류검출
- 오류통보
- 순서나열
- 흐름제어
- 신속 처리 데이터 전송 서비스
- 리셋
- 서비스 선택
- 네트워크 계층관리



## 7. 데이터링크 계층

> 네트워크와 물리 계층 사이

### 7.1 목적

- 네트워크체들 사이의 데이터 교환
- 네트워크체들 사이에 데이터링크 연결을 설정, 유지, 해제하여 데이터(데이터링크 SDU)를 교환
- 노드 대 노드 전달(node-to-node delivery)의 역할

### 7.2 데이터링크 계층의 역할

- 프레임 구성
- 물리주소 지정
- 흐름제어
- 오류제어
- 접근제어

### 7.3 데이터링크 계층의 기능

- 데이터링크 연결의 설정 및 해제
- 링크 서비스 데이터 유니트의 전송
- 데이터링크 연결 종단 확인
- 순서나열
- 오류통보
- 흐름제어



## 8. 물리 계층

> 데이터링크 계층 아래

### 8.1 목적

- 데이터링크체들 사이의 데이터 교환
- 물리적 연결의 설정, 유지, 해제를 통해 하나의 노드에서 다음 노드까지 각 비트를 전송함

### 8.2 물리 계층의 역할

- 물리적 매개체의 인터페이스
- 비트의 부호화와 복호화
- 비트 전송속도
- 비트의 동기화
- 전송회선 구성
- 물리적 구조
- 전송 모드

### 8.3 물리 계층의 기능

- 물리적 연결의 설정 및 해제
- 물리적 서비스 데이터 유니트의 전송
- 물리 계층관리
